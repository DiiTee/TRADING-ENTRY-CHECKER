ATTENTION BRIEFING ARCHITECTURE UPDATE

OBJECTIVE 

Transform Attention Briefing from a proxy-based momentum estimator into a true Attention Intelligence Engine capable of detecting:

- Attention Ignition
- Attention Velocity
- Attention Acceleration
- Attention Persistence
- Attention Quality
- Source Expansion
- Narrative Growth

The goal is to identify attention BEFORE it fully manifests in holder growth, volume growth, and market cap expansion.

Refine the Attention Briefing architecture to account for the reality that free social data sources are not always available.

The system must NOT assume that social data will always exist.

Instead, Attention Briefing should dynamically adapt its confidence, weighting, and scoring based on available data.


CRITICAL REQUIREMENTS

DO NOT REMOVE EXISTING ATTENTION BRIEFING FEATURES.

Preserve:

- Holder Momentum
- Holder Divergence
- Attention Efficiency
- Narrative Strength
- Lifecycle Analysis
- Existing proxy calculations

These remain valuable fallback signals.

Real social attention should ENHANCE and partially blend with existing proxy metrics, not completely replace them.

--------------------------------------------------

CORE DESIGN PRINCIPLE

Real Social Mode should be treated as an enhancement layer.

Proxy Mode should remain fully functional.

The app must continue producing useful Attention Briefings even when social data is unavailable.

FAILURE HANDLING

If one social source fails:

Do NOT automatically enter Proxy Mode.

Attempt to use remaining sources.

Examples:

X unavailable
Telegram available
→ PARTIAL SOCIAL MODE

Telegram unavailable
X available
→ PARTIAL SOCIAL MODE

Reddit unavailable
X + Telegram available
→ FULL SOCIAL MODE

Only enter PROXY MODE when no usable social source data exists.
