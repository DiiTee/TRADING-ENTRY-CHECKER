1. X / Twitter Abstraction Engine Instructions

We will use TwitterAPI.io as our primary Data Ingestion Layer and Apify as our structural fallback. Do not hardcode a single provider; create a provider abstraction wrapper that catches timeouts, handles provider failures gracefully, and fails over dynamically.

Provider Priority:

1. TwitterAPI.io
2. Apify

Search Priority:

1. Contract Address
2. Ticker
3. Coin Name

Contract Address matches should always receive the highest confidence weighting.

Primary Data Source: TwitterAPI.io

Primary Data Source Details:

* Purpose: High-speed, cost-effective chronological tracking.
* Base URL: https://api.twitterapi.io/v1/twitter/search
OR: 
https://api.twitterapi.io/twitter/tweet/advanced_search

* Authentication Method: Pass the API key in the HTTP Header as X-API-Key.

Required Parameters for Quick Scan:

* query: The search term string (Contract Address, Ticker, or Name).
* limit: Set to 15 (or 50 for Deep Scan).
* product: Hardcode to "Latest" (ensures a strictly chronological data slice).

Incremental Re-Scan Parameter Rule:

If a lastScanTimestamp exists for the token in our database, append:

since_time:unix_timestamp

directly to the query string.

Example Query String:

0xABC123... since_time:1717660800

Fallback Data Source: Apify (Tweet Scraper V2)

Fallback Data Source Details:

* Purpose: Auto-triggered secondary circuit-breaker if TwitterAPI.io fails.
* Base URL:

https://api.apify.com/v2/acts/apidojo~tweet-scraper/run-sync-get-dataset-items

* Authentication Method:

Append token to URL query string:

?token=YOUR_APIFY_TOKEN

Payload Blueprint (JSON POST):

{
  "searchTerms": ["YOUR_SEARCH_TERM"],
  "sort": "Latest",
  "maxItems": 15,
  "includeReplies": false
}

4. Analytical Calculations & Memory Layout

Pipe all returned payloads into internal processing structures before database writes.

In-Memory Windowing & Density Calculation

1. Store incoming timestamps inside an in-memory structure or Redis Sorted Set.

2. Calculate:

Time Span Minutes

(Newest Timestamp - Oldest Timestamp) / 60

3. Apply Density Safety Floor:

Time Span Minutes = max(Time Span Minutes, 1)

This prevents unrealistic density spikes when all mentions occur within seconds.

4. Calculate:

Mention Density =
Total Mentions
÷
Time Span Minutes

Classification:

* VERY_HIGH
* HIGH
* MODERATE
* LOW
* VERY_LOW

Example:

15 mentions within 5 minutes

→ VERY_HIGH

---

Time-Weighted Confidence Logic

Calculate:

Author Concentration Ratio Formula

Prevent manipulation by mapping usernames into an internal frequency dictionary.

Formula:

Author Concentration Ratio

=
Mentions From Most Active User
÷
Total Mentions

8 mentions

Ratio:

8 / 15

= 0.53
