STRUCTURE ENGINE REWORK — STRUCTURE RELIABILITY, ENTRY CLASSIFICATION & OHLCV SUFFICIENCY UPGRADE

IMPORTANT

DO NOT remove the Structure Engine.

DO NOT remove:

- Floor calculations
- Shelf detection
- Historical Low calculations
- Safe Entry Range calculations
- Existing Entry Range calculations
- Structural diagnostics

DO NOT rebuild:

- Security Engine
- Attention Engine
- Holder Engine
- Flow Engine
- Dev Reputation Engine
- Audit Engine

DO NOT remove:

- Full Mode (new name for Structure Mode)
- Hint Mode
- Flow Mode

This is NOT a replacement of the Structure Engine.

This is a Structure Reliability and Entry Classification layer that sits on top of the existing Structure Engine.

────────────────────────────
PRIMARY GOAL
────────────────────────────

The current Structure Engine treats many estimated floors as if they have similar confidence levels.

This becomes problematic for:

- Fresh launches
- Early Telegram signal coins
- Coins in active price discovery
- Coins with only a few candles
- Coins with little retrace history
- Coins whose floor is mostly LP-derived

The engine should continue providing Entry recommendations.

However:

The app must first determine:

1. How reliable the current structure is.
2. What type of entry is appropriate.
3. Whether OHLCV is missing OR simply insufficient.

────────────────────────────
KEEP EXISTING MODES
────────────────────────────

Do NOT replace:

- Full Mode (new name for Structure Mode)
- Hint Mode
- Flow Mode

These modes should remain exactly as they are.

The new system should operate ABOVE them.

Conceptually:

ENTRY CLASSIFICATION
↓

18/100

This preserves existing architecture.

────────────────────────────
STRUCTURE RELIABILITY FACTORS
────────────────────────────

Positive Factors:

+ OHLCV available
+ Candle count
+ Coin age
+ Retrace count
+ Shelf count
+ Confirmed support retests
+ Historical low confidence
+ Multiple successful support reactions

Negative Factors:

- Missing OHLCV
- Very low candle count
- Coin age under 2 hours
- No retraces
- No shelf structure
- Low transaction count
- LP-derived floor estimate
- Active price discovery
- Vertical expansion without pullbacks

ENTRY TYPE CLASSIFIER

Before Entry recommendations are displayed:

Classify the market into:

1. DISCOVERY ENTRY
2. MOMENTUM ENTRY
3. STRUCTURE ENTRY

Display in a new panel.

────────────────────────────
DISCOVERY ENTRY
────────────────────────────


Primary trigger:
Structure Reliability < 30

Secondary trigger:
Two or more Discovery conditions below:

- Structure Reliability < 30
- Coin age < 60 minutes
- Candle Sufficiency = Insufficient
- No meaningful shelf
- No meaningful retraces

Display:

ENTRY TYPE:
DISCOVERY ENTRY

Description:

Price discovery is active.
Structure is immature.
Support calculations are low confidence.

Suggested Exposure:

25–50% of normal position size.

Display:

DISCOVERY ENTRY ZONE

Current MC → Current MC + 5%

This is participation guidance.

Not support-based guidance.

Keep existing floor calculations visible.

Add:

Floor Confidence:
Low

Disclaimer:

Estimated floor should not be treated as confirmed support.

────────────────────────────
MOMENTUM ENTRY
────────────────────────────

Trigger when:

Structure Reliability 30–60

AND

Flow remains positive.

Examples:

- Buy pressure positive
- Holder growth positive
- Attention score positive
- No major distribution

Display:

ENTRY TYPE:
MOMENTUM ENTRY

Purpose:

Continuation entry based on flow rather than confirmed support.

Display:

Aggressive Zone:
Current MC → Current MC + 8%

Conservative Zone:
Wait for first pullback.

Avoid Entry If:

- Buy pressure collapses
- Major holder distribution appears
- Dev selling emerges

Keep existing floor visible.

Add note:

Current floor estimate exists but is not yet considered confirmed support.

────────────────────────────
FINAL RULE
────────────────────────────

The purpose of this upgrade is NOT to replace the current Structure Engine.

The purpose is to:

1. Measure structure quality.
2. Distinguish missing data from insufficient data.
3. Classify entry type appropriately.
4. Improve confidence communication.
5. Preserve all existing floor, shelf and entry calculations.

The user should always receive Entry guidance.

The app should simply communicate whether that guidance is based on:

DISCOVERY, MOMENTUM,
or
CONFIRMED STRUCTURE.

────────────────────────────
LP DEPTH RULE
────────────────────────────

LP depth must NOT be used as a primary floor generator.

LP depth may only:

- Reduce confidence
- Act as a fallback diagnostic
- Support risk calculations

LP depth alone should never create a high-confidence support level.

A floor generated primarily from LP assumptions must be labelled:

Estimated Floor

and receive reduced Entry Confidence.


Floor mostly LP-derived should simply apply:
-20 Floor Confidence

It shouldn't directly affect any Entry Type.
