STRUCTURAL ANALYSIS ENGINE IMPROVEMENTS

OBJECTIVE

Improve the engine's ability to distinguish:

- Healthy accumulation
- Re-accumulation before continuation
- Dying shelves
- Distribution shelves
- Dead-cat bounces

while preserving current functionality.

ANTI-DOUBLE-COUNTING RULE

Avoid using the same underlying signal to heavily influence multiple major scores.

Example:

Higher-Low Structure already contributes to:

- HL Ladder Score
- Shelf Type Classification
- Structure Sequence

Do not allow Higher-Low Structure to dominate Demand Return Score as well.

Each major score should measure a distinct aspect of market structure whenever possible.


========================================================
FIX #1 — ACCEPTANCE SCORE NORMALIZATION

Problem:

Current Acceptance Score components are weighted twice.

Example:

densityPart =
(component score × percentage)
then multiplied again by weight.

This causes some components (especially volume concentration) to contribute far less than intended.

Required Change:

Step 1:

Calculate raw component scores:

DensityScore (0–100)
DefenseScore (0–100)
CompressionScore (0–100)
VolumeScore (0–100)
DistanceScore (0–100)

Step 2:

Apply weights:

AcceptanceScore =
DensityScore * 0.35 +
DefenseScore * 0.25 +
CompressionScore * 0.20 +
VolumeScore * 0.10 +
DistanceScore * 0.10

Clamp result to:

0–100

FIX #4 — DEMAND EXPANSION SCORE

Purpose:

Detect whether demand is increasing or fading.

Output:

0–100

Inputs:

Recent Window:
Last 15 candles

Previous Window:
15 candles immediately before that

Primary Method:

If buy/sell data exists:

Buy Count Expansion

Buy Volume Expansion

Measure growth versus previous window.

Scoring:

Large increase = high score

Stable = neutral

Declining = low score


A full breakdown of the Demand Expansion Score: 

Growth >50% = 100

Growth 25–50% = 85

Growth 10–25% = 70

Growth 0–10% = 55

Decline 0–10% = 40

Decline 10–25% = 25

Decline >25% = 10

FIX #5 — TRUE BOUNCE VELOCITY

Problem:

Current implementation measures bounce size.

It does not measure bounce speed.

Required:

Identify:

Shelf Defense Low

Bounce High

Calculate:

BounceMagnitude =
((BounceHigh - ShelfLow) / ShelfLow) × 100

BounceDuration =
Number of candles needed to reach BounceHigh

Velocity =
BounceMagnitude / BounceDuration

Score:

Fast + Large Bounce = 100

Moderate = 75

Slow = 50

Weak = 25

Dead Bounce = 0

Display:

Bounce Magnitude %

Bounce Duration

Bounce Velocity Score
