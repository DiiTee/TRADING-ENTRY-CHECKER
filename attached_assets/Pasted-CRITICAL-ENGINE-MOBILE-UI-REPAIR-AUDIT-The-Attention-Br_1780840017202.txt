PRIORITY 1 — ATTENTION BRIEFING (SOCIAL SCAN ENGINE)

CURRENT SYMPTOMS:

- Popular tokens (Fartcoin, Popcat, Pepe, Pudgy Penguins, Ethereum, etc.) consistently receive F-tier Attention scores.
- Social Scan frequently returns:
  "GetXAPI: Failed to fetch"
- Engine falls into Proxy Mode despite analyzing highly active tokens.

ROOT CAUSE:

The scoring system appears functional. The data acquisition layer is failing.

The application is currently deployed on Vercel and appears to be attempting direct browser-side requests to social APIs.

These requests are being blocked by:

- CORS restrictions
- Network failures
- Provider rate limits
- Invalid API responses

REQUIRED FIXES:

1. Move ALL social API requests off the frontend.

Create a Vercel Serverless Function:

/api/social-scan

The frontend should NEVER directly call:

- TwitterAPI.io
- GetXAPI
- Apify
- Reddit APIs

All requests should pass through the serverless function.

2. Implement provider failover chain:

TwitterAPI.io
↓
GetXAPI
↓
Apify
↓
Graceful Degradation

3. Add explicit status reporting.

Replace generic messages such as:

"Failed to fetch"

with detailed diagnostics:

Examples:

✓ TwitterAPI.io returned 127 posts
✓ GetXAPI returned 42 posts
⚠ GetXAPI Rate Limited
⚠ TwitterAPI.io API Key Invalid
⚠ Reddit Request Timed Out
⚠ CORS Blocked
⚠ Provider Returned Empty Dataset
⚠ No matching posts found

4. Distinguish between:

- No results
- Rate limit
- Invalid key
- Timeout
- CORS
- Network error
- Provider outage

These must never all appear as:

"Failed to fetch"

5. Log all provider responses.

Add debug output:

Provider Used
Request URL
Response Code
Post Count Returned
Search Query Used

6. Verify query generation.

Audit searches for:

Coin Name
Ticker
$Ticker
Contract Address

Example:

POPCAT
$POPCAT
Popcat
Contract Address

The engine should test multiple query formats before concluding that no social activity exists.
